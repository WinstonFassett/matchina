# Live Dev Shims (Package Building for Docs Dev Server)

## Why This Matters (DX Gold)

The mkdist setup enables a **zero-friction dev loop**:
- Save code → browser auto-refreshes
- No manual builds, no typechecks, no restarts
- Just have the dev server running and that's it

This is a **huge differentiator** for AI-assisted development — agents don't need to run builds/typechecks themselves. Save file, see changes. Done.

## Problem
When adding new packages to the Matchina monorepo, the docs dev server can't resolve them with "Cannot find module" errors.

## Root Cause
New packages need proper build configuration and workspace setup to be resolvable by the docs.

## Solution Pattern

### 1. Package Structure
```
packages/new-package/
├── package.json          # Must have correct name and exports
├── build.config.ts       # CRITICAL: Use mkdist builder, not default unbuild
├── src/
│   └── index.ts
└── dist/                  # Generated by build
```

### 2. Root package.json Workspaces
```json
{
  "workspaces": [
    ".",
    "docs", 
    "packages/new-package"  // MUST add here
  ]
}
```

### 3. Build Scripts Integration
```json
{
  "scripts": {
    "build:packages": "npm --workspace @matchina/old-package run build && npm --workspace @matchina/new-package run build",
    "dev:prepare": "npm run build:lib && npm run build:packages"
  }
}
```

### 4. CRITICAL: build.config.ts
**WRONG** (causes "missing package.json files" warnings):
```typescript
export default defineBuildConfig({
  entries: ['src/index'],           // ❌ Default unbuild
  rollup: { emitCJS: true },
})
```

**CORRECT** (matches viz-reactflow pattern):
```typescript
export default defineBuildConfig({
  entries: [
    {
      input: "./src/",
      outDir: "./dist", 
      builder: 'mkdist',            // ✅ Use mkdist builder
      format: "esm",
    },
    {
      input: "./src/",
      outDir: "./dist",
      builder: 'mkdist',            // ✅ Use mkdist builder  
      format: "cjs",
    }
  ],
  declaration: true,
  failOnWarn: false,
  externals: ['react', 'react-dom', 'matchina', 'matchina/react', 'matchina/hsm'],
})
```

### 5. Package Exports
```json
{
  "exports": {
    ".": {
      "node": {
        "types": "./src/index.ts",
        "import": "./src/index.ts"
      },
      "types": "./dist/index.d.ts",
      "import": "./dist/index.mjs", 
      "require": "./dist/index.js"
    }
  }
}
```

### 6. Docs Integration
Add to `docs/package.json`:
```json
{
  "dependencies": {
    "@matchina/new-package": "file:../packages/new-package"
  }
}
```

## Build Process
1. `npm run build:packages` - Builds all viz packages
2. `npm run dev:prepare` - Builds lib + packages (runs before docs dev)
3. Docs dev server can now resolve the package

## Key Learnings
- **ALWAYS** use `mkdist` builder for viz packages (not default unbuild)
- **ALWAYS** add to root `workspaces` 
- **ALWAYS** add to `build:packages` script
- **ALWAYS** add to docs dependencies
- **NEVER** use default unbuild entries - causes package.json issues

## Debugging
- Check build output for "missing package.json files" warnings
- Ensure `dist/index.js` exists (mkdist creates this)
- Verify workspace configuration in root package.json
- Run `npm run dev:prepare` before starting docs dev server
