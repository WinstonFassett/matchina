# Transition Flow: Lifecycle Hooks

All machine types implement the same generic [EventLifecycle](/matchina/reference/interfaces/eventlifecycle) and hook methods. The same patterns work across a [FactoryMachine](/matchina/guides/machines), a [StateMachine](/matchina/guides/state-machine-interface), or a [StoreMachine](/matchina/reference/interfaces/storemachine/). ## Step-by-step

### 0. `api[type](...params)` (optional)

A model layer over the machine. `api[type](...params)` may call `machine.send(type, ...params)`. Optional — it keeps callers out of the event-data layer.

### 1. `send`

Where the event enters the machine. `machine.send(type, ...params)` resolves the change event and starts the transition pipeline.

```ts
send(type, ...params) {
  const resolved = machine.resolveExit({ type, params, from: lastChange.to });
  if (resolved) machine.transition(resolved);
}
```

If a generated API method is present (e.g. `api.doSomething(a, b)`), it is the first frame and delegates to `send`.

### 2. `resolveExit`

Resolves the `to` state for the event `{ type, from, to? }` — looks up the transition handler for the current state and event type and returns the resolved `to` state.

- Return `null` or `undefined` to exit before the lifecycle pipeline runs.

### 3. `transition`

The lifecycle pipeline. Each step below runs in order against the resolved event; `guard`, `handle`, and `before` can each halt propagation.

### 4. `guard`

Is the transition allowed?

- If `false`, the pipeline exits.

### 5. `handle`

First opportunity to process or rewrite the event.

- If returns `undefined`, the pipeline exits.

### 6. `before`

Final hook before the state change is applied. Can still rewrite the event.

- If returns `undefined`, the pipeline exits.

### 7. `update`

Applies the new state.

### 8. `effect`

Runs side effects. By default, this calls `leave` then `enter`.

### 9. `leave`

Called as the previous state is exited.

### 10. `enter`

Called as the new state is entered.

### 11. `notify`

Notifies subscribers of the change.

### 12. `after`

Final hook after the transition completes.

## Early exits

`resolveExit` returning no match, `guard` returning `false`, or `handle`/`before` returning `undefined` will stop the flow. No further steps run.
