## When to Use This Pattern

**Use game state machines for:**
- Turn-based games with distinct phases and rules
- Interactive applications that need to enforce business logic
- Scenarios where actions are only valid in certain states
- Complex conditional logic that depends on current game state
- Multi-step processes that need to track progress and context

**Key benefits:**
- Enforces game rules through state transitions
- Prevents invalid actions by restricting available events
- Clear separation of game phases and their logic
- Easier testing of individual game states
- Simplified debugging of game flow issues
