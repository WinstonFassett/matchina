## When to Use This Pattern

**Use simple state machines for:**
- Systems with a fixed sequence of states
- Processes that follow predictable patterns
- UI components with distinct modes or phases
- Any scenario where state transitions are deterministic and clear

**Key benefits:**
- Eliminates invalid states (can't have two lights at once)
- Ensures proper sequence (can't skip from Red to Green)
- Clear state-dependent behavior and styling
- Easy to test and debug
- Foundation for more complex state patterns

## Key Features

- **Multiple States**: Red, Yellow, and Green with distinct messages
- **Cyclical Transitions**: Proper traffic light sequence using `next` event
- **State-based UI**: Conditional styling based on current state
- **Type Safety**: Typed state data and transitions
