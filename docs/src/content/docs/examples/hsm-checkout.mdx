## When to Use This Pattern

**Use hierarchical checkout for:**
- Multi-step forms with distinct phases and sub-steps
- E-commerce flows where payment is a separate concern
- Wizards where certain steps have their own internal state management
- Complex user journeys that need to maintain context across phases

**Key benefits:**
- Clear separation of concerns between main flow and sub-flows
- Ability to reset or retry sub-flows without losing main context
- Simplified state management for complex business processes
- Better code organization and maintainability
