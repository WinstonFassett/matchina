## When to Use This Pattern

**Use hierarchical combobox for:**
- Complex form controls with multiple interaction modes
- Autocomplete components with keyboard navigation
- Tag editors that need to manage both selection and input states
- UI components that have distinct operational modes with sub-behaviors

**Key benefits:**
- Clear separation between UI mode and interaction logic
- Easier keyboard navigation implementation
- Better accessibility support through structured state
- Simplified testing of individual interaction modes
- Reusable patterns for other complex form controls
